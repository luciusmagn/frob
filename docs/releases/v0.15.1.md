# Autolith 0.15.1

Autolith 0.15.1 refines concurrent-agent presentation and replaces raw
`task.run` argument JSON with readable compact and expanded layouts.

## Child-agent strip

- Added blank rows between the `agents N` heading, child rows, and modeline.
- Indented child rows and aligned their role column using terminal cell widths,
  including at narrow widths and with unequal identifier lengths.
- Removed the redundant `async` label while preserving detached execution
  behavior and visibility.
- Applied the same indentation to overflow summaries without adding an extra
  blank row when detached children remain visible at an idle prompt.

## Task call presentation

- Added a compact `task.run` layout with one aligned summary row per child,
  readable shared context, concise item-specific context, and cell-aware
  assignment truncation.
- Added an expanded layout with complete per-task sections for name, role,
  execution mode, item context, and wrapped assignment text.
- Made the durable `/compact` preference select between those layouts in
  addition to controlling routine tool-result visibility.
- Added safe structured fallbacks for malformed JSON, invalid batches, and
  invalid task items. Task calls no longer expose raw argument JSON, and all
  layouts remain bounded on narrow terminals.

## Task reliability

- Gave each private child conversation its task execution identifier, preventing
  concurrently created children from accidentally sharing a provider thread
  when short conversation identifiers collide across separate artifact roots.

## Compatibility notes

- The compact-view preference format is unchanged.
- Presentation remains ephemeral. Saved task calls and results are unchanged,
  and replay uses the currently selected compact-view setting.
