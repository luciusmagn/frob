# Autolith 0.16.0

Autolith 0.16.0 adds native Skills and Model Context Protocol support while
keeping the two extension mechanisms independent.

## Native Skills

- Added strict `SKILL.sexp` discovery from project, user, and bundled roots,
  with deterministic precedence and bounded diagnostics.
- Added `skill.load` for explicit logical-turn selection. Complete Skill
  instructions are reread from the selected file and injected only into
  request-local context on subsequent requests in that turn.
- Kept `skill.load` calls and results out of durable history, replay, and
  compaction input, and made selection a provider round-trip barrier before
  later actions.
- Added `/skills` and expanded `/context` observability without persisting
  Skill bodies or selection state in conversation history.
- Confined discovery to regular files inside configured roots and retained the
  selected file identity so a same-name replacement cannot be applied
  silently.
- Bounded aggregate discovery input, selected Skill counts, and warning output,
  collapsing excess warnings into one mandatory notice.

## MCP

- Added the separate Mcparen Common Lisp client library and pinned it as an
  Autolith dependency.
- Added negotiated MCP 2025-11-25 and 2025-06-18 over standard input and output
  and Streamable HTTP, including session lifecycle, server notifications,
  resumable event streams, bounded pagination, cancellation, resources,
  templates, prompts, and tools.
- Added strict native `mcp.sexp` configuration, layered user Lisp
  registration, required and optional servers, environment-backed credentials,
  and deterministic provider-safe tool namespaces.
- Bounded each manager to 1,024 retained tools and 8 MiB of exact normalized,
  credential-sanitized, encoded input schemas, allocating required servers
  before optional servers during startup and tool-list refresh.
- Projected initialization metadata to recognized capability presence, removed
  unused server information, and bounded retained server instructions to 2,000
  characters per server.
- Added explicit per-call approval, trusted read-only policy, exact child-agent
  grants, `/mcp`, `mcp.status`, `mcp.refresh`, resource helpers, and prompt
  helpers.
- Kept server instructions request-local and treated server metadata and
  results as untrusted external data.

## Persistence and lifecycle

- Preserved ordered text and image blocks from MCP results through provider
  input and append-only conversation replay.
- Added a process-wide transient-secret guard so checkpoints cannot retain
  provider or MCP credentials.
- Used one exact environment snapshot for MCP launch values, HTTP headers, and
  redaction, and restarted persistent standard-input servers when mapped
  credentials changed or disappeared.
- Made startup, reconnect, and MCP reload clean up partial runtimes and preserve
  the last committed application when replacement construction fails.
- Closed child task runtimes before shared MCP clients and detached inherited
  transports before saving a core.

## Format boundaries

- Skills use Autolith's native `SKILL.sexp` format. Autolith does not load
  `SKILL.md` compatibility files.
- MCP configuration uses Autolith's native `mcp.sexp` format. Codex MCP
  configuration is neither read nor migrated.
- MCP tools that require protocol task execution remain visible in status but
  are not exposed as callable Autolith tools.
