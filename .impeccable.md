## Design Context

### Users

Autolith is used by its technical creator and other terminal-oriented developers
as a live, self-modifying Common Lisp agent. The interface should make runtime
identity and session context immediately legible without displacing useful
conversation scrollback.

### Brand Personality

Cosmic, imposing, and sharp. Autolith should feel like a compact machine with a
distinct identity, not a generic developer dashboard or novelty hacker screen.

### Aesthetic Direction

Use a restrained terminal composition with one memorable mark: an `AL` FIGlet
wordmark in the Cosmic font. Prefer a light-green xterm-256 row gradient when
available, with bold basic-green fallback. Keep session metadata compact and
aligned beside the mark. Give the mark one blank row above it, divide the title
from model and workspace details, keep internal conversation identifiers out of
the header, and place operational warnings below the whole composition. Avoid
generic full-name ASCII banners, rainbow color, purple-blue gradients, glow
effects, card-like boxes, and decorative text that carries no information.

### Design Principles

1. Give identity one strong focal point and keep everything else functional.
2. Use terminal space economically and preserve ordinary scrollback.
3. Align related metadata into a compact, stable reading column.
4. Treat color as progressive enhancement, never as required information.
5. Respect terminal capabilities, `NO_COLOR`, sanitization, and narrow widths.
