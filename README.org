Autolith, or AL, is a small terminal agent that runs inside a Common Lisp
image. It owns its model loop, tools, conversations, authentication, live
mutation, checkpoints, and recovery. It talks directly to the ChatGPT Codex
subscription service and does not launch or bundle the Codex CLI.

The supported target is Linux x86-64 on SBCL. Autolith is dogfoodable, but it
is not a security sandbox: model-generated Lisp runs with the user's privileges,
and the user can grant commands full access for a session.

* Highlights

- A responsive primary-screen terminal interface with ordinary scrollback.
- Multiline editing and reusable completion widgets provided by Clinedi.
- Input remains editable while model responses and tools are running.
- Local PNG, JPEG, GIF, and WebP input with numbered composer labels.
- Persistent, opt-in provider-visible reasoning summaries with =/trace on=.
- Append-only portable conversation files with exact resume commands.
- Workspace and global persistent memories with ranked request-local recall.
- Short workspace agendas with memory links and repository transport.
- Warm in-process fff path, glob, plain, regex, fuzzy, and multi-pattern search.
- Everyday filesystem and shell tools plus named, isolated Lisp REPLs.
- Immutable modified worker images with notes and pristine side-by-side comparison.
- Hash-verified matching SBCL source and Lisp-level implementation instrumentation.
- Live =self.*= mutation with private commits and complete replay scripts.
- Optional =--immutable= startup with inspection-only =self.*= tools.
- Forked checkpoints with per-generation reconstruction artifacts.
- A validated preloaded active image for fast startup and source fallback.

* Install and start

Nix is the recommended installation path. It pins the complete runtime and
keeps it separate from the host system.

#+BEGIN_SRC sh
nix run github:luciusmagn/autolith
#+END_SRC

The conventional Linux x86-64 installer is shorter, but piping network input
into a shell deserves suspicion. Inspect [[https://sh.lambda-symbolics.com/autolith][the script]] first if you use it.

#+BEGIN_SRC sh
curl -fsSL https://sh.lambda-symbolics.com/autolith | sh
#+END_SRC

The binary release contains its own pinned SBCL, matching SBCL source, locked
Lisp dependencies, search library, and private sandbox helper. It needs Git,
OpenSSL, and Bubblewrap from the host. The first launch builds machine-specific
recovery and active images without requiring Quicklisp, a compiler, or a source
checkout. Autolith checks published Git tags periodically and asks before
installing a newer release. Run =autolith --update= to check explicitly.

Authenticate and start a conversation:

#+BEGIN_SRC sh
autolith --auth
autolith
#+END_SRC

If a development checkout has no current fast startup image, the launcher asks
whether to run its bootstrap. Use =autolith --from-source= to deliberately load
the checkout without validating or rebuilding that image.

Enter submits. Shift-Enter, Ctrl-Enter, or Alt-Enter inserts a newline. Ctrl-C
clears a draft or exits from an empty prompt. Up and Down move across visual
input rows before falling
back to history. Tab and Shift-Tab cycle command suggestions. Messages can be
drafted and queued while AL is responding. Type =/help= for commands.

* Repository map

- =src/= contains the active agent in the single =autolith= package.
- =bin/= contains the stable launcher and pinned-runtime adapter.
- =recovery/= contains the minimal source recovery runtime.
- =script/= contains setup, image builders, and repository checks.
- =server/= contains the Common Lisp release server, host builder, and deployment files.
- =tests/= contains the complete regression suite.
- [[file:docs/guide.org][docs/guide.org]] covers setup, operation, tools, persistence, and recovery.

- [[file:docs/release-service.org][docs/release-service.org]] documents the host-built release service.

- [[file:AGENTS.md][AGENTS.md]] defines repository, Lisp, testing, and commit policy.

* Persistent paths

Autolith keeps credentials, preferences, exact command approvals, and static
agent configuration under =${XDG_CONFIG_HOME:-~/.config}/autolith/=. Deferred
inputs, live-mutation journals, and atomic selections belong under
=${XDG_STATE_HOME:-~/.local/state}/autolith/=, including a private Git history
of durable mutation snapshots. Conversations, canonical image commits,
memories, generations, runtimes, matching implementation source, and worker
images live under =${XDG_DATA_HOME:-~/.local/share}/autolith/=. Credentials
never belong in Git, conversation files, memories, journals, manifests, or
saved cores.

Source remains authoritative for clean rebuilds. Saved cores retain exact live
states; private image commits and each generation's =reconstruct.lisp= retain
durable user-specific changes without patching the tracked repository. Missing
canonical replay artifacts are restored from the separate private Git history.

* Development

Source development additionally needs Quicklisp, Rust with Cargo, a C compiler,
CMake, and =pkg-config=. Materialize the locked environment and baseline images
with =./script/bootstrap=.

Run the complete check after every change:

#+BEGIN_SRC sh
./script/check
#+END_SRC

Refresh dependencies and both installed baseline images with
=./script/bootstrap=. Rebuild only the pristine recovery image with
=./script/build-recovery=. See [[file:AGENTS.md][AGENTS.md]] before editing and the
[[file:docs/autolith-minimal-technical-spec.org][technical specification]] before changing architecture.
